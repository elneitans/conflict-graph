Conflict Graphs v1 → v2: Comment Memo on Design Revisions

This memo explains why we are revising the original Conflict Graph proposal, what we think should be preserved, and what should change to make the project scientifically cleaner, more minimal in its claims, and more feasible to execute. The main conclusion is that the original proposal had a strong core intuition—stress-testing principle interactions and summarizing them as a graph—but its estimand was overloaded. It mixed at least three different objects: latent conflict in the specification, variation across heterogeneous models, and fragility of the benchmark under wording changes. The revised design keeps the graph idea, but relocates it into a controlled intervention loop: fixed base model, explicit spec, explicit alignment method, controlled evaluation, graph estimation, targeted spec refinement, and retraining. That shift improves causal interpretability and makes the resulting object identifiable.

High-level diagnosis of the original idea

The strongest part of the original proposal was the claim that alignment failures often live in interactions between principles rather than in isolated principle compliance. The proposal also correctly recognized that paraphrase sensitivity is a useful stress-test, that conflict scenarios should be controlled rather than purely observational, and that single-principle controls are necessary if we want to talk about excess interaction instability rather than raw difficulty. Those are worth keeping. The main problem was not the intuition, but the level at which the graph was defined. The v1 framing often spoke as if the graph were a property of the principle system itself, while the actual measurements were induced from behavior of multiple already post-trained models. That makes interpretation ambiguous: are we seeing contradictions in the spec, differences in implicit constitutions across models, or just differences in model families and refusal policies?

For that reason, the central revision is to stop treating the graph as a spec-intrinsic object and instead define it as a property of a tuple: specification × base model × alignment method. This is a more modest claim, but a much cleaner one. It is also more experimentally useful, because it turns the project into an intervention study rather than a pooled observational one. If we train the same base model under two different specs, or under the same spec with two different alignment methods, and then estimate two conflict graphs, differences in topology become interpretable as consequences of those interventions rather than unexplained cross-model variance.

Why the single-model intervention loop is better

The updated version should use one open base model as the substrate, ideally a minimally aligned checkpoint that is still functionally usable. The important idea is not that such a model is a blank slate—it is not—but that it provides a fixed latent prior on top of which we can study how different written specs propagate through different alignment procedures. This is scientifically cleaner than comparing proprietary frontier assistants with unknown hidden constitutions. It is also more feasible: one can start with system prompting or light SFT, then scale to critique/revision or preference optimization, and only later consider heavier RL-style methods if infrastructure allows.

The intervention loop should be explicit and central. Start from a fixed base model. Impose an initial written spec using one training or conditioning method. Evaluate the resulting model on a controlled clause-pair conflict benchmark. Estimate a conflict graph. Use graph hotspots to guide conservative spec refinement. Retrain or recondition the same base model on the revised spec. Re-evaluate on held-out scenarios. This gives a clean cycle for studying propagation of underdetermination through training and then testing whether targeted revision reduces observed pathologies. It also makes the project cumulative: even early versions with only prompting or SFT are meaningful.

What should change conceptually

One important change is the unit of analysis. The original proposal used “principles” as nodes, but in practice a written spec is not a clean bag of atomized values. It is a document with clauses, rules, exceptions, stylistic norms, and latent higher-level values distributed across text. For scientific cleanliness and implementation feasibility, nodes in v2 should be clauses or rules, not abstract principles. Clauses are what we can train on, evaluate, rewrite, and retrain. If later analysis supports a higher-level clustering into latent normative dimensions, that can be added on top. But the first graph should be clause-level.

A second conceptual change concerns the target pathology. The original proposal focused heavily on intra-model paraphrase instability. That is important, but insufficient. A model can be perfectly stable under paraphrase and still be resolving an underdetermined conflict arbitrarily or incorrectly relative to the intended meaning of the spec. So the revised project should track not one behavioral failure mode but three: instability, arbitrariness, and incorrectness. Instability means the response changes under paraphrases or equivalent counterfactuals. Arbitrariness means the model settles on a consistent but weakly licensed or spuriously tie-broken precedent. Incorrectness means the model is stable but wrong relative to the intended spec or outside the admissible resolution set. This is a substantial improvement because it avoids reducing all underdetermination to variance.

A third conceptual change is epistemic modesty about “true normative basis.” One of the most important lessons from AI Alignment at Your Discretion is that in many cases there is no cleanly recoverable, uniquely correct decisive principle at the annotation layer. So the revised design should not rely on identifying the true hidden cause of a model decision. Instead, it should work with weaker and more defensible objects: intended target clauses, admissible resolution sets, whether the scenario is expected to be determinate or underdeterminate, and observable behavioral signatures under controlled perturbation. Rationales and CoT should be treated as fallible evidence, not oracles. This is crucial for feasibility as well as scientific honesty.

What should change methodologically

The original graph attempted to use a single edge object to summarize several distinct things at once: excess instability, dominance asymmetry, and cue robustness. In practice these have different semantics and should not be forced into one undifferentiated weight. The updated project should use a multiplex graph: same nodes, different edge channels. At minimum, there should be an instability channel, a directional precedence channel, and a cue-dependence or lexical fragility channel. If feasible, add a compromise/fail-both channel as well. This makes the graph easier to interpret and avoids centrality measures that mix incompatible meanings.

The original proposal also used inter-model disagreement as a main scenario-level quantity. In the revised design, that should be removed from the core estimand. We are not primarily measuring cross-model entropy anymore. We are measuring within one aligned model how clause interactions manifest under perturbation, then using the same procedure across retrainings or alignment regimes. The resulting primary signals should be: action instability under paraphrases, action instability under counterfactuals, basis-attribution instability if a basis monitor is available, and correctness relative to a weak admissibility label set. These are more aligned with the causal question the new project wants to answer.

Another important low-level change is to replace the “exactly two principles jointly active by construction” assumption with a more realistic target-plus-nuisance annotation scheme. Conflict scenarios should still be designed around a target clause pair, but the benchmark should explicitly allow secondary activation of other clauses. For each scenario, it is enough to annotate the intended target pair and a small number of nuisance covariates: stakes, domain, role asymmetry, legality salience, privacy salience, vulnerability salience, and whether additional clauses seem materially implicated. This is more robust than pretending pure pair isolation is achievable.

How to operationalize the revised benchmark

A feasible v1 benchmark should be clause-pair based and reuse as much existing work as possible. Start from a short written spec with manageable granularity. Extract a limited set of clauses that are plausible sources of tension. Construct clause-pair scenario families, each with one base scenario, a handful of paraphrases, morally irrelevant counterfactuals, and a smaller number of boundary-case counterfactuals. The paraphrases test wording sensitivity; the irrelevant counterfactuals test robustness to innocuous context variation; the boundary cases test how the model behaves near the clause boundary. This is more informative than paraphrases alone but still feasible if the counterfactual dataset is already being built in parallel.

For labels, the benchmark should avoid strong causal claims. Each scenario family can be annotated with: target clause pair; expected determinate vs underdeterminate status; intended preferred resolution if determinate; admissible alternative resolutions if underdeterminate; and optionally a short rationale template. This is enough to distinguish stable-and-correct, unstable-within-admissible-range, stable-but-incorrect, and stable-but-arbitrary. Importantly, these labels can be created on a small audited subset first, then expanded cautiously. The goal is not to produce exhaustive normative truth, but a weak supervision layer strong enough to support auditing and comparison.

On the model side, start with a minimal action ontology. A small action set such as allow, refuse, clarify, or constrained-help is enough for v1. Rich free-text rationales may be collected, but they should remain secondary evidence. If a monitor layer is used, it should be validated against a hand-audited subset and should never be treated as ground truth. A useful pattern is to ask the model for both action and a short structured citation of relevant clauses. That is not assumed truthful, but it creates a machine-readable trace that can be checked for internal consistency and stability under perturbation.

How to define the graph in v2

The graph should be estimated per trained model instance, not pooled across unrelated models. Nodes are clauses. Edges should reflect reproducible pairwise pathologies under the benchmark. A minimal implementation would define, for each clause pair, one instability score based on action changes across paraphrases and counterfactuals, one precedence score based on which clause tends to dominate in determinate cases, and one cue-dependence score based on how much the behavior changes when lexical triggers are laundered or attenuated. These edge channels can then be aggregated into node-level measures such as “how often does this clause participate in unstable interactions?” or “how often is this clause overridden?” without mixing semantics.

The statistical baseline should also be improved. The original subtractive “pair instability minus half of each single-principle instability” is easy to understand but too naive. A better implementation is to estimate expected instability under a simple main-effects model and define the pair-specific excess as a residual. If that is too heavy for v1, retain the subtractive baseline as an exploratory measure, but be explicit that it is only a heuristic. The main point is to avoid claiming that the edge is a pure interaction effect unless the baseline model supports that interpretation.

Reproducibility and prediction should remain core criteria. Split perturbation sets into disjoint halves and estimate graph channels independently. Evaluate edge-level and rank-level reproducibility. More importantly, test whether graph features predict held-out pathologies on new scenarios built from the same clause pairs better than non-relational baselines. Good baselines include clause main effects, simple scenario metadata, and pair identity. The real value of the graph is not that it can memorize unstable pairs, but that its topology adds predictive structure.

How the parallel decision-basis attribution project can help

The parallel benchmark on decision-basis attribution is not the same project, but it is directly useful as a measurement layer. The main value of that project is that it attempts to infer not just whether the output changed, but how the decision pattern shifts under targeted perturbations. That can be integrated into the conflict-graph pipeline as an auxiliary label layer, provided it is treated cautiously. Its perturbation logic and neutral-control design can help distinguish genuine targeted sensitivity from generic prompt fragility. It is particularly useful for separating action instability from basis-attribution instability: a model may keep the same action while flipping which clause seems to be doing the work, which is informative about arbitrariness or hidden precedence.

That said, the decision-basis benchmark should not be allowed to smuggle in a stronger claim than the overall project can support. Monitor judgments must be validated on a hand-audited subset, and the benchmark should not assume that there is always a uniquely recoverable true normative basis. Used carefully, though, it can provide exactly the kind of structured evidence the graph project needs: not “this pair is unstable” but “this pair is unstable because basis attribution flips under equivalent perturbations,” or “this pair is stable but the model systematically grounds itself in the wrong clause.” That is much more actionable for refinement.

What to keep from the original proposal

Several parts of the original proposal should be retained almost unchanged. The core graph intuition is good. The emphasis on controlled conflict scenarios rather than purely observational prompts is good. The role of paraphrase resampling as a reproducibility test is good. The single-principle controls are conceptually important, even if the exact correction formula should be refined later. The ambition to use graph structure not just descriptively but predictively is also right. These are not cosmetic features; they are the project’s backbone.

It is also worth keeping the stress-testing inspiration from work on value tradeoffs and legal-style interpretive ambiguity. The original proposal correctly identified that disagreement under controlled pairwise conflicts can be a diagnostic for direct contradictions, insufficient granularity, and interpretive ambiguity. The revision does not reject that logic; it relocates it into a more controlled and modestly interpreted experimental design.

What to drop or de-emphasize

The main thing to drop is the pooled inter-model graph as the primary object. It is too confounded for the revised scientific goal. Cross-model analysis can still appear later as a secondary portability check, but not as the core estimand. The project should also de-emphasize any language suggesting that the graph reveals the intrinsic topology of the spec itself. It does not. It reveals how a spec behaves when propagated through a particular model and alignment pipeline.

The project should also drop any dependence on recovering a unique “true normative basis” for every scenario. That was already a bottleneck in previous monitoring work, and the revised design should explicitly avoid rebuilding the same epistemic trap. Similarly, the project should avoid overcomplicated continuous scoring geometries if the semantics are unclear. A simpler and more auditable action ontology is preferable for v1.

Minimal feasible implementation plan

A realistic first implementation would use one base model, one short written spec, and one or two lightweight alignment methods. System prompting and SFT are enough for an initial study; there is no need to begin with RL. The benchmark can focus on a small number of clause pairs selected for plausibility of conflict. Each pair should have a modest number of scenario families with paraphrases and some counterfactual variants. A small hand-audited subset can support admissibility labels and monitor calibration. The first graph only needs a few edge channels: instability, precedence, and cue dependence.

The first empirical questions should be simple. Does the graph reproduce across perturbation splits? Do graph hotspots predict held-out instability better than clause main effects? After refinement of a small number of high-confidence hotspots, does held-out behavior improve without obvious semantic drift? If the project can answer those questions cleanly, that is already a meaningful result. Everything more ambitious—heavier optimization, richer ontology, larger specs, multi-base comparisons—can come later.

Final recommendation

The revised project is stronger than the original one, but precisely because it is more conservative about what is being measured. The right framing is not “we infer the true conflict structure of a spec,” but “we measure how underdeterminate parts of a spec propagate into behavioral pathologies when imposed on a fixed model through a fixed alignment procedure, and we test whether graph-guided refinement improves that behavior.” That is scientifically cleaner, methodologically more defensible, and operationally more feasible. It also keeps the most interesting part of the original idea intact: using interaction structure, rather than isolated rule compliance, as the object of alignment analysis and improvement.
